Extended Story Visualization:

A key-frame sequence approach towards video generation from unstructured text.

Until today, many methods in video generation [21] focus mainly on the motion consistency rather than the long-term context of the generated video. In other words, the model is trained to improve the smoothness of each frame with respect to the prior.

Video generation example. C. Vondrick, H. Pirsiavash, A. Torralba, NIPS 2016

Although this has yielded good results, if we foresee machines creating more relatable visual content that takes into account context, this becomes a non-trivial task. As an alternative, it is possible to separate the task into two steps, first to create models capable of generating specific initial a target frames to relate a story in a visual manner, and then to use a motion-focused module that can illustrate each piece of the story in a keyframe-by-keyframe interpolation.

 

 

This key frame-like generation is proposed as a relatively new task by [1], with the name of Story Visualization. Where the task is treated as a sequence conditional generation problem, using story embeddings as the prior for the hidden state of the generator, and creates a new image for each individual sentence given in the story. This approach showed significantly good results where it achieved context-aware generation taking into account character and background distribution given a written story, nonetheless, it showcased a limitation in which each generated image was paired to a given sentence. Thus, in the aim of long-term video generation, we considered relevant to propose an extension to the method in which the model would receive an unstructured text entry and then be able to select the relevant features to generate the best number of images that illustrate the story.

Model overview

Paragraph to image-Sequence model (P2iSeq).

As shown in the image above and similar to the StoryGAN, the model comprehends a sequential Generative Adversarial Network (GAN) with two discriminators (the image discriminator for local consistency and story discriminator for global consistency). For the generation, it receives the encoded story coming from a DistilBERT pre-trained encoder as the first hidden vector ( ), and a set of stacked vectors compounded by a random vector sampled from a standard normal distribution ( ) and the embeddings from the sentences selected by the extractive summarizer. At training time the generator is updated in two independent loops, first taking into account the image discriminator output and the second one with respect to the story discriminator.

In the case of the proposed architecture, the key contribution relies in the implementation of a selective module that is able to choose the most relevant features of the story in order to avoid repetition in the image generation and maintains the context of the whole story, as well as a network capable of dealing with variable length stories coming from the generator and fixed-length ground truths.

Story Encoder

For the story encoder module, we considered using BERT by Google. BERT is a stack of transformers pre-trained on the Wikipedia corpus. It comes in two sizes, BERT base and BERT large. The base model has 12 transformer layers and the large model has 24. The BERT model, however, is too large and requires too many resources for training as well as inference. After much deliberation, we chose to use DistilBERT in our story encoder. DistilBERT is a smaller version of BERT. By using DistilBERT we reduce the size of a BERT model by 40%, while retaining 97% of its language understanding capabilities and being 60% faster. This is achieved by leveraging knowledge distillation in the pre-training phase. Knowledge distillation is a compression technique in which a compact model, the student, is trained to reproduce the behaviour of a larger model, the teacher, or an ensemble of models.

 

We started by taking a pre-trained DistilBERT model and trained (fine-tuned) it on text from the Pororo-SV dataset. We did this to improve the encodings generated for sentences in the dataset (i.e. it learns that Pororo is a name). Next, we used this model to encode the stories in our dataset. The text encodings are then passed to the recurrent generator. We also added a linear layer that maps the 768 dimension vector embeddings from DistilBERT into a 228 dimension vector corresponding to two 124-dimension vectors for the mean and variance respectively. Furthermore, the embeddings were stacked to a one-hot encoded vector that represents the characters present in the story to help the model identify them in the generation process.

 

Extractive Summarization

This summarization approach was inspired by [3], where it takes advantage of the similarity of the sentences and removes sentences where the information is already represented by other parts in the text. The process starts with a neural coreference resolution of the personal pronouns, in which every time the reference is found the model replaces the mention with its correspondent nominal pronoun. For the sake of this resolution problem, a pre-trained neural model based on [23] is used and then input the resolved sentences to the before-mentioned DistilBERT encoder to return the respective embeddings. The resolved embeddings are then clustered and the summarizer outputs the embeddings that are closer to each of the cluster centroid, the proportion of clusters generated is a hyperparameter tuned over the experimentation.

 

Coreference weights visualization. (*Image taken from HuggingFace Co. demo. )

Training

In order to train the generator and both discriminators in an end-to-end pipeline, we use two independent loops, the first one to allow local consistency between the generated image and text input, and the second one for global consistency in which it's taking into account the set of generated images conforming a story and the written story itself when computing the losses. Image Discriminator

Essentially, for the updates corresponding to the image loss ( ) we used a binary cross entropy loss that summed up the the outputs of the discriminator with respect to the generated and ground truth images over the corresponding story. To allow the simultaneous evaluation of the image distribution and its correlation with the input story sentence in the discriminator, it concates the encoded raw image output of a convolutional layer with the sentence and story embeddings to determinate whether it came from the generator or not.

​

Story Discriminator

Similar to the image discriminator, the computed story loss ( ) utilizes a binary cross entropy loss from the story discriminator outputs, but instead of taking into account every image-sentence embeddings pair separately, it stacks and averages every story related set of vectors to then be fed into the discriminator.

This process was particularly challenging to implement due to the fact that the ground truths and the generated length of the image stories would not necessarily be of the same length. The key idea of this discriminator is to evaluate the story embedded mean information remains the same between ground truths and generated samples even when they had different length.

For the final term of the optimization objective, we made use of a KL Divergence penalty between the story embeddings distribution and a normal standard distribution to avoid the mode-collapse issue in the image generation process.

Dataset

To correctly evaluate the context consistency in our proposed model, we used the Pororo dataset originally used for video question answering, but it has been modified to fit the story visualization task by considering the description for each video clip as the story’s text input by [1]. The dataset contains 15,336 description-story pairs, where 13,000 pairs are used as training, the remaining 2,336 pairs for testing.

Pororo-SV dataset main characters.

Running previous models on this dataset like ImageGAN does not generate consistent image sequences; for instance, the generated images switch from indoors to outdoors randomly. Additionally, the characters’ appearance is inconsistent in the sequence of images (e.g. Pororo’s hat).

Experiments

We started by implementing a baseline from previous related work, in this case, a modified version of the available StoryGAN implementation for the CLEVR dataset in PyTorch, in which we first wanted to validate the capabilities of the model in the Pororo dataset and replicate the results shown in the CVPR paper. And then we proceeded to evaluate the proposed model in two stages, fixed-length generation and the summarized version with variable-length generation. Both experimentation pipelines are described in the next sections and training curves throughout the tuning process are listed in the Appendix.

 

Fixed-length generation

In our first experiment, we implemented a DistilBERT story encoder to verify if it will provide enough context to the generator so the output images make sense. We noticed that as the training progressed, the images generated became clearer and made more sense. In most of the generated sequences, we saw that the DistilBERT embeddings passed enough context information to accurately identify the characters present in the input story as well as their exact positions with respect to each other and other objects in the scene. One such sequence is described below, where the model generates all the characters in their correct positions but it has not been able to detect the bed should be in the generated distribution.

P2iSeq fix-length image generation showing the correct identification of main characters.

Additionally, the color distribution of the scene, as well as the background environment (outdoors/indoors), are generally well illustrated by the model as seen in the below examples.

P2iSeq generated visual stories matching the context distribution of the ground truth counterparts.

Summarized generation

Once we achieved considerable good results in the fixed-length image generation experiments in contrast to the StoryGAN baseline, we continued to set the experimentation pipeline in the variable length approach proposed with the Extractive Text Summarizer module. This added a higher level of complexity to the optimization problem, the GAN training became more unstable and made the job easier for the discriminator since it was harder for the generator to create a interpolated image distribution that covered the full story information with less amount of images.

Summarized image generation example showing the correct sentence selection.

As shown in the figure above, we weren't able to reach the desired image quality due to the available amount of computational resources. But it is noticeable the model was capable of identifying the underlying distribution in terms of background (i.e. the framed sequence shows a snowy background in the first image even when none of the selected sentences mentioned it) and in terms of characters selection; since the summarizer chose the features illustrating the most relevant ones in the story.

 

Conclusions

As a result of this project, we successfully implemented a model capable to identify key features from a given written story and summarize it visually. Although, at the current stage it is unable to reach a reliable image quality, our approach is capable of learning the underlying context distribution of the Pororo dataset, and we conclude it needs further fine-tuning to stabilize training in order to reach the desired image quality. Throughout the experimentation, we noticed the generator became a slower learner with respect to the story discriminator, since during the summarization process it needed to learn the image-to-image distribution at the same time it had to interpolate or try to create images that shown the same amount of information in less amount of images.

Future work

Another complicated parameter to tune in the training process, was the ratio of summarized senteces of the extractive module. Hence, we believe adding a soft-relaxation to the sentence selection process would help the model identify the correct amount of sentences to generate at training time, as it would backpropagate the error when generating the incorrect amount of images for a given story.

Soft selection mask in summarization module to allow backpropagation.

And additional to investigating this proposed approach, we consider it'd be interesting to validate the model in a broader datase with ground truths counterparts are selected from longer written stories.

 

Appendix

Related Work

Story Visualization is a relatively new task with only one paper (StoryGAN [1]) published in early 2019. It falls into the broad categorization of generative tasks such as text-to-image generation, video generation, style transfer, and image editing. Tasks that are relevant for story visualization are conditional text-to-image transformation [5, 12, 19, 18], and text-to-video [13, 9] or image-to-video generation [7, 16, 17]. There are many things that are key to these two tasks as well. For example, in text-to-image generation, an important objective is to understand longer and more complex input text. This has been explored in dialogue-to-image generation [20], where the input is a dialogue session and not a single sentence. Similarly in video generation, the current approaches only generate short videos with no scene changes. To model the motion features of the objects, the researchers disentangle dynamic and static features for motion and background [21].

Several other tasks which are also related to story visualization are story image retrieval from a pre-collected training set instead of generating images [15], a “cut and paste” technique for cartoon generation [8]. The opposite task of story visualization is visual storytelling where the output is a paragraph describing the input sequence of images. For visual storytelling task text generation models or reinforcement learning [11, 14, 10] are used.
